## Template Instructions

**How to Use This Template**:

1. **Copy this template** at the end of each week (Friday afternoon or Monday morning)
2. **Update the header** with current week number and date range
3. **Fill in KPI Snapshot** using data from [Beta Tester Tracker](beta-tester-tracker.md) and [QA Execution Log](qa-execution-log.md)
4. **Summarize progress** in completed/in-progress/not-started sections
5. **Highlight blockers** in the Top Issues section
6. **List key decisions** that need stakeholder input
7. **Include feedback quotes** to humanize the data
8. **Plan next week** with clear objectives and tasks
9. **Update Exit Criteria Progress** to show how close we are to production
10. **Share with stakeholders** via email or Slack

**Frequency**: Weekly (recommended: every Monday morning)

**Audience**: Project Lead, Dev Lead, Product Lead, QA Lead, Stakeholders

**Distribution**: Email + Slack + Saved to `docs/release/status-reports/`
